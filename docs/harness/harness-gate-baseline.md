# 合并/发版 Harness 证据门禁基线（R4）

## 目标

建立统一的 merge/release 证据门禁，确保所有关键变更在合并与发版前具备可追溯验证证据。

## 核心规则

- 无验证证据不合并。
- 无验证证据不发版。
- 证据不可复核视为未验证。
- 例外放行必须留痕并限定有效期。

## Merge 门禁（必须满足）

1. PR 已关联 Issue、Spec、Contract/Rule。
2. 必需验证项已执行并给出证据：
- Build/Typecheck
- Unit Test
- Smoke
3. 阻断项为 0：
- 未解决 BLOCKER = 0
- 不存在边界越界且未回写真源
4. 风险与回滚路径明确。

## Release 门禁（必须满足）

1. 候选版本已冻结，范围边界明确。
2. 最小 smoke 清单通过并可复核。
3. 风险项、回滚口径、发布证据已记录。
4. 与本次发布相关的 BLOCKER 全部关闭。

## 证据标准

每项证据至少包含：

- Command/入口
- Result（PASS/FAIL/N/A + 原因）
- Evidence（日志片段、截图或报告链接）
- Conclusion（是否满足门禁）

## 阻断条件

命中任一条件即禁止合并或发版：

- 必需验证项失败
- 必需验证项缺失证据
- 证据不可复核（链接失效、日志缺关键上下文）
- 边界冲突未解除
- 回滚路径不可执行

## 例外审批与留痕

仅允许在紧急止损场景申请例外。

例外必须记录：

- 申请人、审批人
- 例外原因与影响面
- 风险接受声明
- 补齐证据的截止时间
- 到期补救结果

未留痕的例外一律无效。
