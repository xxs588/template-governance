# Harness 阶段计划

## Phase 0：工程秩序

验证内容：每次变更至少能编译/构建通过。

| 检查项 | 作用 |
|--------|------|
| 编译/构建 | 项目可正常构建 |
| 静态分析 | 代码风格与潜在问题 |
| 单元测试 | 核心逻辑覆盖 |

执行方式：本地手动执行，PR 中提供证据。

## Phase 1：契约验证

验证内容：API 实现与契约一致。

| 检查项 | 作用 |
|--------|------|
| 契约格式合法 | lint 工具验证 |
| 契约一致性 | 实现端点/字段与契约对齐 |
| Smoke 脚本 | 核心链路冒烟 |

## Phase 2：交付闭环

验证内容：关键用户路径端到端可用。

| 检查项 | 作用 |
|--------|------|
| E2E 测试 | 关键路径自动化验证 |
| 发版门禁 | build + test + E2E |
| 部署验证 | 一键启动验证 |

将本项目的具体构建/测试命令写入各仓库的 `AGENTS.md`。
