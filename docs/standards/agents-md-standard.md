# AGENTS.md 编写标准

## 目标

让 Agent 在仓库内行为可预测、可约束、可验证。

## 必填章节

1. 项目目标与非目标
2. 技术边界（可改/不可改）
3. 开发命令（构建、测试、运行）
4. 代码规范（命名、类型、分层）
5. 验证方式（最小验收命令）
6. 文档回写要求

## 写作原则

- 短：优先 1~2 屏可读
- 准：只写当前有效规则
- 新：每次重大改动同步更新
- 可执行：每条规则都可落实到操作

## 反例

- 把团队共识写成抽象口号
- 复制大段外部文章无项目映射
- 规则冲突但未给优先级
